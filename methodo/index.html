Comment ça marche

Une routine,
pas un humain

Le brief est produit chaque lundi matin par une routine Claude Code distante. Aucun humain ne lit ou réécrit avant publication — c'est volontaire, pour garder la production rapide, régulière, et sans biais d'embellissement.

Le workflow en 7 étapes

Du cron au push

01

Déclenchement

La routine Claude Code distante est planifiée via le cron 0 6 * * 1 (UTC) — soit 08:00 Europe/Paris l'été, 07:00 l'hiver. L'infrastructure Anthropic spawn un agent dans une sandbox isolée, totalement indépendante de mon PC.

02

Clone du repo

L'agent clone leomarty1/veille-IA dans son workspace. Il lit le brief précédent (briefs/<date>.html) et le registre briefs/data.json pour comprendre l'historique.

03

Récolte des sources

Pour chacun des 5 acteurs, l'agent fait des WebSearch ciblés avec filtre temporel (last 7 days), puis WebFetch sur les 1 à 3 meilleurs hits. Sources primaires seulement — pas de second-hand, pas de tweets, pas d'agrégateurs.

Voir la liste détaillée des sources par acteur.

04

Filtrage

L'agent jette : partenariats sans impact technique, levées de fonds < 5 B$, tribunes/opinions sans news concrète, repackagings de features existantes, posts marketing pur.

Il garde : releases de modèles, nouvelles features/SDK, changements pricing notables, 1 ou 2 papers majeurs max, acquisitions stratégiques.

05

Scoring

Chaque item retenu reçoit un tag :

🎯 Lynxter   impact direct workflow Lynxter — Claude Code, Cowork, MCP, plugins, agents, génération de docs, voix/TTS. Réservé aux gros impacts. 🛠 Utile   à connaître pour anticiper questions client ou changements produit majeurs. · Info   culture IA, pas d'impact pratique immédiat.

06

Rédaction

L'agent compose : TL;DR (3 bullets), Implications Lynxter (2-3 paragraphes actionnables, sous le prisme support 3D + automation + skills custom), Synthèse (3-5 phrases), puis les items par acteur dans l'ordre fixe Anthropic → OpenAI → Google → Meta → Mistral → Autres.

Règle dure : zéro hallucination. Tout item doit avoir une URL source réellement fetched. Si la semaine est calme sur un acteur, c'est écrit explicitement — pas de remplissage.

07

Publication

L'agent génère index.html (dernier brief en home) + briefs/<date>.html (archive permanente), met à jour briefs/data.json et la page d'archive, puis git commit + push origin main. GitHub Pages redéploie automatiquement en 1-2 min.

Cadre éditorial

L'angle Lynxter

Le brief n'est pas un agrégateur neutre. Il est scoré et commenté du point de vue de Léo Marty, responsable support technique chez Lynxter. Le contexte qui informe les décisions de tagging et le contenu des "Implications Lynxter" :

Stack quotidien

  • Claude Code (CLI) — sessions multi-projets, support en parallèle
  • Claude Cowork — gestion des workflows non-dev (skills, plugins)
  • MCPs : Slack, Outlook, Gmail, GitHub, Microsoft Learn
  • Skills custom : lynxter-support, onboarding-client, veille-ia

Activités cœur

  • Support technique imprimantes 3D industrielles (S300X, S600D)
  • Diagnostic matériaux (PU, silicone, PA, PETG, etc.)
  • Formation client + onboarding
  • Construction de skills/plugins pour automatiser le support

Veille active sur

  • Solutions IA souveraines / EU / on-prem (clients industriels)
  • Voix française pour tutos clients (TTS)
  • Agents long-running pour automation tickets support
  • Outils d'orchestration multi-MCP

Sous le capot

Infrastructure

  • Routine : Claude Code remote trigger (cloud Anthropic)
  • Modèle : Claude Sonnet 4.6
  • Hébergement : GitHub Pages (gratuit, statique)
  • Code source : github.com/leomarty1/veille-IA

Front-end

  • HTML/CSS/JS vanilla — aucune dépendance build
  • Typo Archivo (proxy GTAmerica Expanded) + Roboto + Outfit — Google Fonts
  • Logos acteurs : Simple Icons (CC0)
  • Charte graphique Lynxter v3.1

Conventions de fichiers

  • index.html — toujours le dernier brief
  • briefs/YYYY-MM-DD.html — un fichier par brief
  • briefs/data.json — registre des briefs (date, items_count, highlights)
  • briefs/index.html — archive avec recherche
  • acteurs/index.html — profils des acteurs surveillés
  • methodo/index.html — cette page
