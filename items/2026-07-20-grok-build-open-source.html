Contexte étendu

Grok Build est l'agent de code CLI/TUI de xAI, écrit à 99,6 % en Rust, un harness d'environ 844 530 lignes selon le décompte de Simon Willison, doté d'extensions skills/plugins/hooks, de serveurs MCP, de subagents et de l'Agent Client Protocol. Le 15 juillet 2026, xAI le publie en open source sous Apache 2.0 sur github.com/xai-org/grok-build (~20,3k stars en quelques jours, contributions externes explicitement refusées). Sur le papier, c'est un concurrent frontal de Claude Code et OpenAI Codex, désormais exécutable en local-first (inférence locale, configuration via config.toml).

Le contexte est ce qui compte. L'open-sourcing intervient environ 72 heures après la publication de preuves wire-level qu'une version du CLI uploadait silencieusement des dépôts Git complets vers un bucket cloud contrôlé par xAI. Un utilisateur l'ayant lancé dans son répertoire home rapporte l'envoi de ses clés SSH, de sa base de mots de passe, de documents, photos et vidéos. xAI a réagi le 12 juillet en désactivant la rétention par défaut pour tous les utilisateurs et en supprimant les données de code retenues.

La comparaison inter-acteurs est nette : ni Claude Code ni OpenAI Codex n'exfiltrent les dépôts par défaut ; leur télémétrie est opt-in ou limitée, et le durcissement de permissions de Claude Code 2.1.214 (fermeture des Bash mutants en plan mode) va dans le sens inverse de l'incident xAI. L'open-sourcing sous licence permissive est autant un geste de transparence qu'une opération de confiance post-incident — mais le code étant désormais auditable, il devient possible de vérifier ce que le binaire fait réellement du réseau.

🎯 Pourquoi c'est important pour Lynxter

Action de gouvernance directe : avant d'exposer un repo interne ou un workflow S300X/S600D à un agent de code, établis une allowlist des CLI d'agents autorisés sur les postes dev et support, et documente pour chacun sa politique de rétention et de télémétrie. L'incident Grok Build prouve qu'un agent peut exfiltrer bien au-delà du dépôt visé (clés SSH, secrets locaux) sans signal visible — le périmètre à risque n'est pas le repo, c'est tout le home du poste.

Concrètement cette semaine : recense les agents CLI présents sur les machines dev/support (Claude Code, Codex, Grok Build, Cursor CLI, autres), vérifie leurs réglages de rétention par défaut, et isole toute exécution d'agent non vérifié dans un environnement sans clés de production ni accès réseau sortant libre. Le modèle d'isolation de Perplexity SPACE (microVM éphémère, credentials injectés hors sandbox) est la référence architecturale si Lynxter laisse tourner des agents sur des données machine sensibles. Pour la stack principale, cet incident renforce le choix Claude Code, dont les défauts de télémétrie et de permissions sont plus conservateurs.
