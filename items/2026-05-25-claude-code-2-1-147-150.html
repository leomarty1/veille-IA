Contexte étendu

Anthropic enchaîne quatre releases Claude Code en trois jours. 2.1.147 (21 mai) renomme /simplify en /code-review — la commande lance un sous-agent qui passe en revue le diff courant et propose améliorations / refactor / catch bugs, en miroir du Code Review GA annoncé au Code with Claude SF du 6 mai. Les sessions épinglées en background restent maintenant actives au repos (avant : timeout). Auto-updater plus fiable. 2.1.148 (22 mai) est un hotfix : un bug de la 2.1.147 faisait retourner exit code 127 sur les commandes Bash chez certains utilisateurs.

2.1.149 (22 mai) est la release feature de la semaine. /usage ventile maintenant ce qui consomme tes limites en quatre catégories : skills, subagents, plugins, et coûts par serveur MCP. Précieux pour identifier en un coup d'œil quel plugin Cowork bouffe ton budget tokens. La vue détail de /diff est maintenant navigable au clavier (j/k, PgUp/PgDn, Space, Home/End) — fin du scroll souris obligatoire. Les listes de tâches GFM en Markdown (- [ ] / - [x]) sont rendues avec les cases à cocher dans les transcripts. Côté Enterprise : nouveau paramètre allowAllClaudeAiMcps qui autorise tous les connecteurs MCP du cloud claude.ai à être chargés (au lieu d'une allowlist par connecteur). Corrections de sécurité PowerShell, git worktrees, Bash. 2.1.150 (23 mai) : infra interne, rien de visible utilisateur.

Les sorties parallèles de la semaine du 23-25 mai (à venir vendredi) ajoutent aussi terminalSequence dans le JSON output des hooks (desktop notifications, window titles, bells sans terminal contrôlant), CLAUDE_CODE_PLUGIN_PREFER_HTTPS pour cloner les plugins via HTTPS au lieu de SSH (utile en environnement sans clé SSH GitHub), ANTHROPIC_WORKSPACE_ID pour scope un token de workload identity federation à un workspace spécifique, et claude agents --cwd <path> pour filtrer la liste des sessions par répertoire. La rewind menu gagne "Summarize up to here" qui compresse le contexte ancien en gardant les derniers tours intacts.

🛠 Pourquoi c'est utile pour Lynxter

Aucune feature blocker, mais trois améliorations à exploiter dans les routines Cowork. Le /usage par catégorie est l'outil qui te permet enfin de répondre "pourquoi mon Max 20x est-il à 60 % de quota un mardi soir ?" — tu vois immédiatement si c'est le plugin diagnostics S300X qui spawn trop de subagents, le MCP server qui se reconnecte à chaque requête, ou simplement un script qui boucle. Compte 30 minutes pour passer ton dashboard Cowork actuel en revue, identifier le top 3 consommateurs et arbitrer (désactiver, throttler, ou ré-implémenter en plus efficient).

Le renommage /code-review demande une mise à jour de doc interne — si tes opérateurs ont des aides-mémoire ou des scripts d'onboarding qui mentionnent /simplify, ils faut les passer en /code-review. Le rewind "Summarize up to here" change le flow long-running : sur une session support qui accumule 50+ tours (ticket complexe avec aller-retour client + diagnostic + résolution), tu peux maintenant compresser le tout en un summary court tout en gardant les 5 derniers tours intacts — la session continue avec un contexte propre. Idéal pour les Routines longues comme la veille-IA elle-même quand elle franchit le seuil de compaction auto.
