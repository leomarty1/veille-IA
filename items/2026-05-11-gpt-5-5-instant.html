Contexte étendu

OpenAI remplace GPT-5.3 Instant par GPT-5.5 Instant comme modèle par défaut de ChatGPT pour les utilisateurs payants et gratuits. Trois axes principaux d'amélioration : −52.5 % d'hallucinations sur prompts à enjeu (médical, juridique, financier — les domaines où l'erreur coûte cher) ; 81.2 sur AIME 2025 (math compétition lycée) contre 65.4 sur 5.3 Instant — saut massif ; et −30 % de longueur de réponse pour exprimer la même chose (signal que le modèle est devenu plus dense).

Deux features produit notables au-delà du modèle : recherche dans historique conversations, fichiers et Gmail intégrée (concurrence Connectors d'OpenAI et Claude Cowork) ; et — surtout — Memory sources affichées dans ChatGPT, c'est-à-dire que tu peux voir d'où vient chaque info mémorisée par le modèle, et la supprimer/corriger individuellement. C'est une réponse directe à la critique sur l'opacité de la mémoire ChatGPT.

GPT-5.3 Instant reste accessible 3 mois pour les utilisateurs payants avant retrait — fenêtre de transition standard chez OpenAI.

🎯 Pourquoi c'est important pour Lynxter

Lynxter n'est pas sur GPT (stack Claude) donc l'impact direct est limité. Mais deux points utiles :

1. Anticiper les questions client. Les utilisateurs ChatGPT vont remarquer le saut, et certains vont demander à Léo "pourquoi tu n'utilises pas GPT-5.5 ?". Réponse pré-mâchée : Claude Opus 4.7 garde l'avance sur le coding (87.6 % SWE-bench, GPT-5.5 Instant n'est pas évalué sur ce bench), et Claude Code offre une intégration plus profonde au workflow Lynxter via Routines + Cowork.

2. Memory sources est une vraie bonne idée. La transparence sur l'origine de chaque info mémorisée est probablement la direction que toutes les memory features vont prendre. Anthropic Memory en beta n'a pas encore cette transparence ; à surveiller si elle arrive.
