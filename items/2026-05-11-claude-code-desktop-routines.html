Redesign Claude Code desktop + Routines

Extrait du brief Anthropic consolide, Mistral arme — 2026-05-11

Contexte étendu

Anthropic publie le 14 avril deux annonces couplées qui redéfinissent l'expérience Claude Code pour les utilisateurs intensifs. La première : redesign complet de l'app desktop Mac et Windows autour du paradigme "many things in flight, you in the orchestrator seat". Concrètement, ça veut dire une sidebar multi-sessions qui permet de garder plusieurs conversations vivantes en parallèle, un workspace drag-and-drop pour réorganiser visuellement les sessions, et un terminal + éditeur intégrés directement dans l'app sans avoir à jongler avec VS Code en parallèle.

La seconde annonce est Routines (research preview) — un système qui bundle un prompt, un repo cible et des connectors MCP en une configuration exécutable. Une routine peut être déclenchée sur schedule cron, via appel API, ou sur événement GitHub (push, issue créée, PR ouverte). C'est effectivement un cron + GitHub Actions natif piloté par agent, qui s'intègre dans la session Claude Code mais tourne en cloud Anthropic indépendamment de la machine locale.

Rollout immédiat sur Pro, Max, Team et Enterprise. Pas de tier gratuit pour Routines à ce stade.

🎯 Pourquoi c'est important pour Lynxter

Routines est exactement le pattern qui sous-tend les plugins Lynxter : lynxter-support et veille-ia sont déjà conceptuellement des "prompts + contextes + connectors déclenchables sur événement". Jusqu'à présent, l'orchestration se faisait manuellement (Léo lance le skill quand il en a besoin) ou via du scheduling bricolé. Avec Routines, ça devient natif et industrialisable.

Cas d'usage concrets immédiats : lundi matin → récap des tickets support de la semaine dans Slack ; nouvelle issue GitHub priorité haute → pré-diagnostic automatique posté en commentaire ; chaque jour 18h → check status pipeline onboarding clients. Tout ce que Léo fait aujourd'hui à la main qui suit un rythme régulier devrait être migré vers Routines dans les semaines qui viennent.

Côté UI : la sidebar multi-sessions du desktop redesigné fit parfaitement le pattern Léo (support ticket en parallèle d'un script en parallèle d'un brief). C'est l'un des principaux pain points résolus de l'usage quotidien.
