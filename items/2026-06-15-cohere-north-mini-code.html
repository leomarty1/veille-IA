Contexte étendu

Le 9 juin 2026, Cohere lance North Mini Code, premier modèle de sa nouvelle famille « North » de coding agents. C'est un MoE de 30B paramètres totaux dont 3B actifs par token, publié sous licence Apache 2.0 avec un contexte de 256K tokens. Son argument d'accessibilité : il tourne sur un seul H100 en FP8, ce qui le met à portée d'un déploiement local ou souverain, sans dépendance à un cloud propriétaire. La cible explicite est l'agentic coding — résolution autonome d'issues, navigation et édition multi-fichiers dans un dépôt.

Côté performance, North Mini Code est évalué via le harness mini-SWE-Agent et affiche 80,2 % pass@10 et 61,0 % pass@1 sur SWE-bench Verified (la métrique pass@10 autorise plusieurs tentatives par tâche, pass@1 une seule — c'est ce dernier chiffre qui se compare aux scores frontière mono-tentative). Cohere revendique qu'il bat des modèles open-source jusqu'à 123B paramètres tout en offrant jusqu'à 2,8× le débit de sortie de Devstral Small 2, son concurrent open-weights le plus direct sur le créneau du code. Le modèle est aussi benchmarké sur SWE-bench Pro et Terminal-Bench v2.

Le positionnement est clair : un modèle compact, rapide et déployable localement, qui ne vise pas le sommet absolu du classement (réservé aux Fable 5, GPT-5.5, Opus 4.8 propriétaires) mais le meilleur rapport capacité/empreinte en open weights. Dans un mois marqué par la suspension réglementaire de Fable 5, l'argument « pas de fournisseur qui peut te couper » prend un relief particulier : un modèle Apache 2.0 téléchargé et exécuté sur ton matériel n'est révocable par personne.

🛠 Pourquoi c'est utile à savoir pour Lynxter

North Mini Code est la brique « IA de code souveraine » la plus crédible de la semaine pour Lynxter. Pour des clients industriels qui ne peuvent pas envoyer leur code ou leurs données vers un cloud US — défense, aéronautique, médical réglementé — un coding agent open-weights qui tient sur un H100 sur site répond à un besoin concret. 61 % pass@1 ne rivalise pas avec Opus 4.8, mais c'est largement suffisant pour de l'assistance de code encadrée, de la génération de helpers, ou de la première passe sur des tâches bien cadrées, le tout sans aucune fuite de données hors du périmètre client.

Action : un benchmark comparatif sur une tâche Lynxter réelle. Monte North Mini Code sur un H100 (ou une instance équivalente) et oppose-le à Devstral sur une tâche représentative — migration d'un set de scripts de calibration, génération de gcode-helpers, ou résolution d'issues sur un dépôt interne. Mesure le taux de réussite pass@1, le débit et la VRAM consommée. S'il tient à 60-80 % de réussite sur du matériel que tu contrôles, tu disposes d'un argumentaire RFP « IA sur site, non révocable » à présenter aux clients sensibles — un différenciateur que la suspension de Fable 5 rend d'autant plus tangible.
