Contexte étendu

Mistral livre Medium 3.5 — un modèle dense de 128B paramètres qui fusionne instruction-following, raisonnement, et coding dans un seul modèle (pas de variantes Instant/Thinking comme chez OpenAI). Score 77.6 % sur SWE-bench Verified, ce qui le place dans la zone très haute des modèles non-frontière — à 10 points de Claude Opus 4.7 (87.6 %), mais largement devant tous les autres open-weights publics.

Le plus important : disponible en open weights sous licence MIT modifiée. La modification autorise un usage commercial libre, avec restrictions sur la réexportation et l'usage par des entités sanctionnées. C'est le meilleur modèle open-weights pour le coding agent à ce jour, prêt pour self-hosting industriel.

En parallèle : Vibe Remote Agents — équivalent de Claude Code Routines + Codex Agents côté Mistral. Permet de spawner des sessions de coding asynchrones depuis le CLI ou depuis Le Chat. Intégration profonde avec Mistral Studio (l'éditeur web Mistral) et Le Chat Entreprise.

🎯 Pourquoi c'est important pour Lynxter

L'alternative française open-weights crédible qu'on attendait. Si un client Lynxter (typiquement défense, aéro, médical) demande un setup IA souverain européen on-prem, Mistral Medium 3.5 est maintenant la réponse. 77.6 % SWE-bench c'est largement assez pour 90 % des cas d'usage support technique.

Trois axes d'action concrets :

1. Préparer un argumentaire commercial "souverain EU" qui mentionne Mistral comme alternative possible. Utile dans les RFP où la souveraineté est un critère.

2. Tester en sandbox via l'API Mistral (à $0.4/$2 par M tokens — moins cher qu'Opus 4.7) pour évaluer la qualité réelle sur des tickets support typiques Lynxter.

3. Vibe Remote Agents à benchmarker — si la qualité est comparable à Routines, on a un fallback EU pour les workflows agentiques sensibles (cf. prédiction n°1).

"Mistral Medium 3.5 atteint 77.6 % sur SWE-bench Verified en licence open weights — un premier benchmark de cette qualité avec un usage commercial libre."
