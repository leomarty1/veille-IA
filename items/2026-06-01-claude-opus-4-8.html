Contexte étendu

Anthropic ship Claude Opus 4.8 le 28 mai 2026, exactement 41 jours après Opus 4.7. Le rythme — 4.5 → 4.6 → 4.7 → 4.8 en six mois — confirme une cadence trimestrielle d'incréments majeurs, sans encore basculer sur Claude 5 (rumeurs Mythos toujours en preview interne). Les scores benchmarks officiels positionnent Opus 4.8 quasiment à égalité avec GPT-5.5 sur SWE-bench Verified (88.6 % vs 88.7 %) — 0.1 pt sous le leader. Mais Anthropic met le focus ailleurs : SWE-bench Pro 69.2 %, soit +4.9 pts vs Opus 4.7 (64.3 %), et 10.6 pts d'avance sur GPT-5.5 (58.6 %). Le Pro évalue des tâches longues, multi-fichiers, mal spécifiées — beaucoup plus représentatif du travail réel de support, audit firmware ou migration de codebase que le Verified.

Anthropic annonce qu'Opus 4.8 bat GPT-5.5 sur au moins 12 benchmarks, notamment l'agentic coding (SWE-Bench Pro 69.2 % vs 58.6 %) et le computer use (OSWorld-Verified 83.4 % vs 78.7 %). Sur GDPval-AA (un benchmark interne d'Anthropic combinant raisonnement général et utilité), Opus 4.8 monte à 1890 Elo, soit 121 pts devant GPT-5.5. Anthropic met aussi en avant des améliorations mesurables sur l'alignement : honnêteté (refus calibrés, moins de hallucinations sur les chiffres et URLs), résistance aux jailbreaks classiques, transparence quand le modèle ne sait pas. Mike Krieger (CPO) parle d'un "near-Mythos level alignment" — référence au modèle preview interne que des rumeurs non officielles situent au-dessus de 93 % SWE-bench, mais qui reste à plusieurs mois d'une release publique.

Le pricing standard reste inchangé à $5 input / $25 output par M tokens, ce qui rend l'upgrade trivial : tu actualises tes Routines Cowork et Claude Code, point. Le vrai changement opérationnel concerne le fast mode : variante de Opus qui tourne à 2.5× la vitesse standard pour 2× le prix par token. Sur Opus 4.7, le fast mode coûtait $30/$150 — économiquement inviable pour la plupart des workloads. Sur 4.8, fast mode est à $10/$50, soit 3× moins cher. Combiné aux Dynamic Workflows (lancées le même jour), le fast mode devient l'option par défaut pour les sous-agents parallèles non-critiques. Disponible immédiatement via l'API Messages, Claude Code, Cowork, et tous les surfaces Claude. Aucune migration nécessaire : le model ID claude-opus-4-7 bascule en alias vers 4.8 sur les plans gérés (Pro, Max, Team, Enterprise) ; l'API garde les deux IDs accessibles.

🎯 Pourquoi c'est important pour Lynxter

Opus 4.8 est l'upgrade le plus low-friction du semestre. Pricing inchangé, qualité supérieure, fast mode économiquement viable, alignement renforcé. Pour ton stack Cowork actuel (génération brief veille-IA, scan tickets support, classification logs S300X), aucune action n'est requise — Anthropic bascule progressivement les sessions vers 4.8. À toi par contre de profiter du nouveau fast mode : les Routines qui produisent du contenu volumineux mais ne demandent pas la qualité absolue Opus (résumés de tickets, classification, déduplication, brief d'arrière-plan) gagnent à passer en fast mode dès cette semaine. Test cette semaine : prends une Routine Cowork existante qui prend > 5 minutes wall-clock, force --model claude-opus-4-8 --fast, mesure latence + qualité subjective (note 1-10) + coût total. Si la dérive qualité reste < 10 %, bascule définitivement.

Le saut sur SWE-bench Pro (69.2 %) est le signal opérationnel. Le Pro mesure des tâches longues, mal spécifiées, multi-fichiers — exactement ton terrain : "trouve pourquoi le profil PEI 90A produit des warpings sur S600D avec ce slicer" ou "migre tous les scripts de calibration Cowork de l'ancien API v2 vers v3". Anthropic est désormais à 10.6 pts d'avance sur GPT-5.5 sur ce benchmark — un gap qui n'est plus rattrapable par optimisation prompt côté concurrent. Si tu hésitais encore à creuser l'usage Claude Code Pro pour les audits firmware ou les migrations de scripts métier, cette release est le déclic. Concrètement : lance un projet pilote sur une migration scriptes test (par exemple, conversion d'un set de Python 3.9 vers 3.12 sur la base de calibration), exécute en parallèle avec Codex (GPT-5.5 backend) sur le même set, compare les résultats. L'écart de qualité devrait être visible immédiatement sur les tâches multi-fichiers.

L'amélioration sur computer use (OSWorld 83.4 %) ouvre un nouveau pattern. Avec Computer Use de Claude couplé à Opus 4.8, l'agent peut désormais cliquer dans des apps desktop avec une fiabilité supérieure à GPT-5.5 (+4.7 pts). Pour Lynxter, ça veut dire : potentiel de faire piloter par un agent les vieux outils Windows métier (Cura, PrusaSlicer, propriétaires fournisseurs filaments) qui n'ont pas d'API moderne. À tester en environnement isolé d'abord — démarre un cas d'usage simple comme "ouvre PrusaSlicer, charge ce STL, exporte le gcode avec ces réglages" et mesure le taux de réussite sur 20 itérations. Si > 80 %, tu as un nouveau pattern d'automation pour les workflows de production où l'API n'existe pas.
