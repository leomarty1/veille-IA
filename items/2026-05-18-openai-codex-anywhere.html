Codex from anywhere — dev sur mobile en preview

Extrait du brief Claude Code multi-agent, OpenAI monte en voix — 2026-05-18

Contexte étendu

OpenAI lance le 14 mai "Work with Codex from anywhere" : Codex est désormais accessible depuis l'application ChatGPT iOS, en preview. Le principe est le suivant — l'app se connecte à la machine distante où Codex tourne (laptop personnel, Mac mini dédié, environment remote managé) et charge l'état live de la session. Tu vois tous les threads actifs, tu sais lesquels bloquent sur ton approbation, tu peux inspecter les derniers outputs.

Depuis le téléphone, les actions disponibles sont : review des outputs (diffs, résultats de tests, screenshots de l'interface en cours de dev), approbation des commandes que Codex veut exécuter, changement de modèle mid-session, démarrage de nouveaux threads. La sécurité est maintenue côté machine : les fichiers, credentials, permissions et setup local restent sur la machine distante. Seuls les outputs remontent en temps réel sur le téléphone. Les updates incluent captures d'écran, output terminal, diffs, résultats de tests et demandes d'approbation.

Cette feature est disponible en preview dans l'app ChatGPT iOS pour les abonnés payants. OpenAI ne donne pas de date de GA mais précise qu'une version Android suit. La feature est complémentaire à Codex desktop, qui reste l'interface principale — le mobile est positionné comme un outil de supervision "en mouvement", pas de démarrage de projets.

🛠 Pourquoi c'est à connaître pour Lynxter

Le signal marché est le plus important ici. OpenAI est en train de valider que la supervision d'agents de coding depuis mobile devient un use case standard. Claude Code a déjà les Remote Control sessions et les Routines — mais pas encore de mobile app dédiée pour superviseur des sessions background. Si la feature Codex mobile prend, Claude Code va probablement répondre dans les prochaines releases 2.x. C'est utile à anticiper pour les futurs workflow multi-sessions Lynxter : la supervision à distance depuis le terrain (atelier d'impression, démo client) deviendra naturelle.

Pour le workflow Lynxter immédiat, la pertinence reste limitée : tes plugins et ton contexte Lynxter sont dans Claude Code, pas dans Codex. Switcher vers Codex implique de reconfigurer tout le contexte, les MCPs custom et les permissions. En revanche, c'est un bon argument de démonstration si un client ou prospect te demande "est-ce qu'on peut superviser les agents IA depuis un téléphone" — la réponse est maintenant oui, chez OpenAI au moins, et Claude Code y travaille de son côté.
