Contexte étendu

La fenêtre couvre cinq releases de Claude Code. La plus structurante, 2.1.178 (15 juin), refond les agent teams : les outils TeamCreate et TeamDelete disparaissent ; avec CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1, chaque session a désormais une équipe implicite et on spawne des coéquipiers directement via le paramètre name de l'outil Agent — plus d'étape de setup. Surtout, elle introduit une nouvelle syntaxe de règles de permission Tool(param:value) qui matche les paramètres d'entrée d'un outil avec wildcard * : on peut par exemple écrire Agent(model:opus) pour interdire les sous-agents Opus (et donc plafonner le coût d'un run multi-agent). Les .claude/ imbriqués sont mieux gérés (le plus proche du répertoire de travail gagne en cas de collision), et l'auto-mode évalue désormais les spawns de sous-agents avant lancement.

2.1.181 (17 juin) ajoute /config key=value pour régler n'importe quel paramètre depuis le prompt (interactif, -p, Remote Control), un opt-in sandbox.allowAppleEvents sur macOS, une variable CLAUDE_CLIENT_PRESENCE_FILE pour couper les notifications push quand on est à la machine, et fait passer le runtime Bun bundlé en 1.4. 2.1.183 (19 juin) est la release sécurité : l'auto-mode bloque les commandes destructrices non demandées — git reset --hard, git checkout -- ., git clean -fd, git stash drop quand tu n'as pas demandé à jeter le travail local, git commit --amend sur un commit non créé par l'agent dans la session, et terraform destroy/pulumi destroy/cdk destroy hors du stack explicitement visé. S'y ajoutent un avertissement quand le modèle demandé est déprécié ou auto-mis à jour (y compris en frontmatter d'agent) et un réglage attribution.sessionUrl pour omettre le lien de session dans les commits/PR.

Les deux autres releases sont des correctifs de fiabilité. 2.1.179 (16 juin) préserve les réponses partielles lors d'un drop de connexion mid-stream (au lieu d'une erreur brute), corrige le scroll molette sous WSL2 (Windows Terminal et VS Code) et un cas où une glob denyRead/allowRead sur une grande arborescence rendait la session inutilisable sous Linux. 2.1.185 (20 juin) ajuste le timing et le texte du hint de stall (« Waiting for API response · will retry in … » après 20 s de silence au lieu de 10 s). Pris ensemble, le fil directeur de la semaine est clair : plus de garde-fous par défaut (auto-mode, permissions paramétriques, garde sur les modèles dépréciés) et une orchestration multi-agent plus simple à câbler.

🎯 Pourquoi c'est important pour Lynxter

L'auto-mode durci est une assurance directe pour une équipe qui fait tourner Claude Code sur des repos réels. Le blocage par défaut de git reset --hard, git clean -fd et consorts évite la classe d'incidents la plus coûteuse : un agent qui, en voulant « nettoyer », efface du travail non commité sur le repo firmware ou les profils de calibration. C'est gratuit (mise à jour de version) et ça change le profil de risque de l'usage agentique en interne. Action : épingle Claude Code à ≥ 2.1.183 sur les postes de l'équipe support/firmware et vérifie que l'auto-mode est actif sur les repos sensibles.

La syntaxe Tool(param:value) est le levier de contrôle de coût qui manquait aux runs multi-agents. Le brief veille-IA et les audits firmware de Lynxter reposent sur des sous-agents ; pouvoir écrire une règle Agent(model:opus) pour forcer les coéquipiers sur un modèle moins cher (ou réserver Opus à l'orchestrateur) borne la facture d'un run qui fan-out. Combine-la avec les agent teams simplifiés (paramètre name) pour câbler un workflow « un sous-agent par acteur » sans setup. À faire : auditer les règles de permission du repo Lynxter (.claude/settings) pour ajouter des plafonds explicites par paramètre.

Le warning sur modèle déprécié tombe à pic vu la cadence de retraits Anthropic. Avec Sonnet 4 et Opus 4 retirés de l'API la même semaine, une Routine qui épinglait un ID mort se serait cassée silencieusement ; Claude Code prévient désormais sur stderr (y compris pour les modèles fixés en frontmatter d'agent). Profite-en pour passer en revue les IDs de modèle codés en dur dans les Routines Cowork et les agents .claude/ de Lynxter, et bascule vers des alias maintenus (Sonnet 4.6, Opus 4.8) plutôt que des versions datées.
