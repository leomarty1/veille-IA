Contexte étendu

Le 9 juillet 2026, Meta Superintelligence Labs sort Muse Spark 1.1 en API payante (Meta Model API, preview publique, US d'abord) — et c'est une rupture doctrinale : premier modèle Meta sans poids ouverts, abandon de la ligne Llama open weights au profit d'un produit frontière commercial. Le modèle est un raisonneur multimodal orienté agentique, contexte 1M tokens, avec un effort de raisonnement ajustable par requête.

Sur les benchmarks tool-use, il prend la tête : MCP Atlas 88,1 devant Claude Opus 4.8 (82,2) et GPT-5.5 (75,3), et JobBench 54,7 contre 48,4 pour Opus 4.8 et 38,3 pour GPT-5.5. Il n'est en revanche que 3e sur le coding et le multimodal — le positionnement est clairement « agent qui orchestre des outils » plutôt que « meilleur codeur brut ».

Le pricing est l'argument de rupture : 1,25 $ input / 4,25 $ output par M tokens, soit environ un quart des tarifs Anthropic et OpenAI, avec une API OpenAI-compatible, 20 $ de crédits offerts et un bref listing sur OpenRouter. Le prix est ~60 % sous la promo Sonnet 5 (2 $/10 $). Contrepartie majeure : c'est une closed API sans option de self-host, donc une dépendance fournisseur totale — l'inverse exact de ce que la doctrine Llama promettait.

🛠 Pourquoi c'est utile à savoir pour Lynxter

Muse Spark 1.1 est un concurrent direct de Sonnet 5/Claude Code sur les workflows agents à orchestration MCP — précisément le terrain de la stack Lynxter. Sa domination sur MCP Atlas (88,1 vs 82,2 pour Opus 4.8) et JobBench en fait un candidat sérieux à benchmarker avant tout arbitrage, d'autant que le prix casse la grille (~1/4 des tarifs Anthropic).

Mais deux réserves conditionnent tout usage industriel : (1) closed API sans self-host signifie dépendance fournisseur totale, sans repli souverain ni exécution on-prem — un point rédhibitoire pour des clients industriels sensibles à la localisation des données ; (2) les scores tool-use sont des benchmarks publics, à valider sur un vrai enchaînement MCP Lynxter (doc S300X/S600D, tickets support, connecteurs internes). Action : monter un test A/B Muse Spark 1.1 vs Sonnet 5 sur un workflow agent existant, en mesurant le taux de succès des enchaînements d'outils et la fiabilité, pas seulement le coût. Tant que la dépendance closed API n'est pas levée, ce reste un point de comparaison, pas un candidat de production.
