Contexte étendu

OpenAI publie Codex CLI v0.142.0 en stable le 22 juin 2026. La fonctionnalité structurante est les budgets de tokens de rollout configurables : le tracking se fait par thread d'agent, avec des rappels du budget restant et un abandon automatique du tour quand le budget est épuisé. C'est un garde-fou de coût appliqué à la granularité de l'agent — utile pour borner la dépense d'un agent multi-étapes qui pourrait sinon partir en boucle coûteuse.

Deuxième primitive : les modes de délégation multi-agents, réglables au niveau thread et turn — disabled (pas de sous-agents), explicit-request-only (délégation seulement sur demande explicite), proactive (l'agent décide de déléguer). OpenAI converge ainsi vers la même primitive de délégation hiérarchique que les agent teams et sous-agents imbriqués de Claude Code (couverts les semaines précédentes), mais avec un curseur de contrôle explicite sur le degré d'autonomie de la délégation.

Le mode web search indexée permet des recherches live tout en restreignant l'accès direct aux pages aux URLs approuvées côté serveur — un compromis entre fraîcheur de l'information et contrôle de ce que l'agent peut effectivement fetcher. S'ajoutent : la redemption de crédits via /usage, la navigation /plugins en catégories Curated/Workspace/Shared, et des corrections (TUI Linux, résilience aux déconnexions, chargement de plugins, persistance de threads, startup/logging). Pas d'annonce officielle fetchable ; faits via releasebot.io et releases.sh.

🛠 Pourquoi c'est utile à savoir pour Lynxter

Les budgets de tokens par rollout sont la fonctionnalité à benchmarker contre Claude Code. Pour Lynxter, qui borne déjà ses coûts Cowork, un garde-fou de budget par thread d'agent est exactement ce qu'il faut pour industrialiser des agents support sans risque de dérive de dépense. Claude Code n'expose pas (encore) un équivalent aussi explicite au niveau du rollout — c'est un point à surveiller dans le changelog Claude Code, et un argument à intégrer si Lynxter compare les deux harnais pour un déploiement à grande échelle où le contrôle de coût par agent est critique.

Les modes de délégation explicites recadrent le choix de harnais agent. Le curseur disabled / explicit / proactive donne un contrôle fin sur l'autonomie de la délégation — utile pour un agent support où l'on veut éventuellement interdire le spawn de sous-agents (mode disabled) sur des tâches sensibles, ou au contraire l'autoriser sur des audits longs. C'est la même logique que les sous-agents imbriqués de Claude Code, et la convergence des deux écosystèmes vers la délégation hiérarchique avec garde-fous confirme que ce pattern est devenu le standard pour l'orchestration multi-agents. Action : si Lynxter teste Codex en second source, comparer la qualité de la délégation proactive vs les agent teams de Claude Code sur un audit firmware multi-fichiers.

Pas de bascule de stack, signal concurrentiel. Lynxter reste sur Claude Code ; Codex v0.142 ne change pas ce setup. L'intérêt est de garder une lecture précise de l'écart de capacités entre les deux harnais agentiques — ici, OpenAI prend une avance ponctuelle sur le contrôle de budget par agent et la délégation paramétrable, qu'Anthropic est susceptible de rattraper dans les semaines suivantes vu la cadence du changelog Claude Code (9 versions en 4 jours cette semaine).
