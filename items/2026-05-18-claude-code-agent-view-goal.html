Contexte étendu

Claude Code 2.1.139, sorti le 11 mai 2026, introduit deux fonctionnalités qui changent fondamentalement le rapport au travail parallèle. Agent View (research preview) est un tableau de bord CLI unique qui liste toutes les sessions Claude Code actives, chacune avec un statut sémantique en temps réel : working (Claude calcule), blocked (attend ta réponse), done (complété). Tu ouvres la vue avec claude agents, tu envoies une session en arrière-plan avec /bg, tu démarres une nouvelle session background avec claude --bg "...". La spacebar inspecte le dernier état sans interrompre, Enter ou flèche droite pour reprendre le fil complet.

La commande /goal est la nouveauté la plus structurante. Tu définis une condition de complétion en langage naturel — "Faire passer tous les tests de la suite S300X", "Générer et commiter le brief sans erreur de linting" — et Claude travaille en autonomie sur autant de tours que nécessaire jusqu'à atteindre l'objectif. Un overlay live affiche le temps écoulé, le nombre de turns et les tokens consommés. /goal fonctionne en mode interactif, en mode -p (headless) et en Remote Control.

La release 2.1.139 complète avec d'autres ajouts utiles : /scroll-speed pour ajuster la vitesse de scroll dans les transcripts, claude plugin details pour inspecter l'inventaire d'un plugin et son coût projeté en tokens par session — utile pour dimensionner les plugins Cowork avant de les déployer sur les plans payants. Navigation dans les transcripts améliorée : {/} pour sauter entre prompts utilisateur, v pour afficher/masquer le panneau de shortcuts.

🎯 Pourquoi c'est important pour Lynxter

Agent View règle enfin le problème de la supervision multi-tâches dans Cowork. Jusqu'ici, jongler entre le brief veille-IA, un script de dev et une session support impliquait plusieurs terminaux ou une gestion manuelle fastidieuse. Avec Agent View, tu ouvres une seule vue et tu sais en un coup d'œil quelle session bloque sur toi — le signal le plus utile en pratique. Le workflow devient : lance 2-3 sessions background au démarrage de journée, travaille en interactif sur ce qui nécessite ta présence, réponds aux blocked quand ils apparaissent dans la liste.

Le /goal command est la pièce manquante pour tes skills long-running. La routine veille-IA telle qu'elle tourne aujourd'hui nécessite des supervision points intermédiaires. Avec /goal "Générer le brief complet de la semaine, commiter et pusher sur main", Claude peut enchaîner fetch → filter → rédaction → génération HTML → git push sans attendre ta validation à chaque étape. La condition de complétion est vérifiable (le commit existe sur main, le CI est vert) — Claude sait quand il a fini. Pareil pour les workflows support S300X/S600D qui impliquent diagnostic → recherche base de connaissance → draft réponse → escalade si nécessaire.
