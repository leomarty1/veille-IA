Contexte étendu

Le 11 mai, Perplexity rend Personal Computer disponible pour tout le monde sur Mac via l'application desktop mise à jour. Personal Computer permet à l'agent Perplexity d'opérer directement sur la machine locale : édition de fichiers locaux, navigation web via Comet, computer use (contrôle de l'interface graphique), orchestration vocale. L'agent voit l'écran, peut ouvrir des applications, remplir des formulaires, et enchaîner des tâches complexes en autonomie. C'est une extension directe du Computer (anciennement "computer use" en preview) au desktop Mac.

En parallèle, Comet — le navigateur IA de Perplexity — est lancé sur iOS cette semaine, complétant le déploiement sur toutes les plateformes majeures : iOS, Android, Mac, et Windows. Comet intègre désormais les préférences mémorisées de l'utilisateur dans son comportement de navigation automatique — si tu lui dis "toujours ouvrir les PDFs dans l'aperçu natif", il se souviendra et appliquera la règle dans chaque session. Un détail notable : Perplexity a choisi Claude Opus 4.7 comme orchestrateur par défaut pour Computer, remplaçant leur précédent modèle maison pour les tâches de raisonnement complexe.

Côté finance, Perplexity ouvre cette semaine une beta US/Canada permettant de lier ses comptes financiers à l'interface web. Les utilisateurs peuvent accéder à un "portfolio page", poser des questions sur leurs positions et mouvements, et demander des analyses à l'agent. Ce n'est pas encore de l'exécution d'ordres — c'est de la lecture et de l'analyse — mais c'est un pas significatif vers un agent financier personnel. Le Computer pour finance professionnelle (accès données marchés, analyse de rapports) suit la même trajectoire.

🛠 Pourquoi c'est à connaître pour Lynxter

Le choix de Claude Opus 4.7 comme orchestrateur par défaut dans Perplexity Computer est un signal marché fort : quand un concurrent choisit ton modèle préféré pour ses tâches les plus critiques, ça valide le choix de stack. Sur le fond, Personal Computer de Perplexity et Remote Control de Claude Code adressent le même besoin (superviser un agent qui opère la machine) mais depuis des positions différentes — Perplexity vient du search/assistant grand public, Claude Code vient du dev. Les deux convergent vers le même paradigme d'agent desktop autonome.

Pour Lynxter en pratique : si un client ou prospect te demande "est-ce qu'on peut automatiser la supervision de l'imprimante depuis n'importe quel device", la réponse est maintenant claire — oui, plusieurs solutions existent (Perplexity Computer, Claude Remote Control, Codex mobile OpenAI). Le marché valide le use case. Ça renforce l'argument pour déployer ce type de supervision chez des clients industriels qui ont des opérateurs en déplacement.
