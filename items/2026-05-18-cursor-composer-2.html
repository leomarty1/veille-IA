Contexte étendu

Cursor lance Composer 2, un modèle de coding spécialisé entraîné pour atteindre des performances frontier-level sur les tâches de développement, à une fraction du coût des modèles généralistes. Le pricing est marquant : $0.50/M tokens en input et $2.50/M en output pour le mode standard, $1.50/M input et $7.50/M output pour le mode fast (plus intelligent). À titre de comparaison, Claude Opus 4.7 est à $15/M output et GPT-5 encore plus cher — Composer 2 cible une parité de qualité sur les tâches coding pour un coût 5-6x inférieur. Le modèle est disponible directement dans Cursor et utilisable via l'API Cursor pour des intégrations custom.

Deux autres nouveautés majeures accompagnent la release. Build in Parallel : Cursor peut maintenant identifier les parties indépendantes d'un plan et les exécuter simultanément via des sous-agents asynchrones — l'équivalent de l'Agent View de Claude Code 2.1, mais intégré directement dans l'IDE. Cursor dans Microsoft Teams : en mentionnant @Cursor dans n'importe quel channel Teams, on peut déléguer une tâche à un cloud agent qui choisit automatiquement le bon repo et le bon modèle, lit tout le thread pour le contexte, implémente et crée une PR pour review.

La release du 7 mai avait déjà introduit une nouvelle expérience de review de PR et des "quick-action pills" pour les workflows courants. L'ensemble forme une stratégie cohérente : Cursor monte en puissance sur la collaboration d'équipe (Teams, PR review) et sur l'autonomie (parallel agents, Composer 2) pour cibler des équipes de dev, pas seulement des développeurs solo.

🛠 Pourquoi c'est à connaître pour Lynxter

Composer 2 fixe un nouveau benchmark prix/performance pour le coding IA. Si tu as des tâches de dev volumineuses où Claude Opus 4.7 commence à peser sur les coûts, Composer 2 via l'API Cursor est une référence à tester. La question n'est pas "Cursor ou Claude Code" — ce sont deux produits différents (IDE vs CLI agent) — mais Composer 2 via API peut devenir intéressant pour des générations de code en batch où le coût token est le facteur limitant.

L'intégration Teams est le signal marché le plus important : les éditeurs de coding IA ciblent maintenant les équipes corporate via les outils de communication existants. Si Lynxter a des clients qui demandent comment intégrer l'IA dans leurs workflows Teams, Cursor en est maintenant un exemple concret et livrable. À surveiller si Anthropic répond avec une intégration Teams native de Claude — probable dans les prochaines semaines.
