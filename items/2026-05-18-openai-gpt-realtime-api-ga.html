Contexte étendu

OpenAI passe la Realtime API en général availability le 14 mai, accompagnée de gpt-realtime — son modèle speech-to-speech le plus avancé à ce jour. Les améliorations portent sur la compréhension des instructions complexes (moins de dérive en cours de conversation longue), la précision du tool calling en contexte vocal, et la naturalité du speech généré. La latence perçue est réduite, avec un meilleur handling des interruptions et des changements de sujet.

Les nouvelles capacités de l'API sont les plus structurantes. Support MCP distant : les serveurs MCP peuvent maintenant être connectés à une session Realtime API, ce qui signifie qu'un agent vocal peut accéder à des données tierces (CRM, base de connaissance, historique tickets) en temps réel pendant un appel. Inputs image : l'agent peut recevoir des captures d'écran ou des images envoyées pendant la session et y réagir vocalement. Appels SIP : l'API accepte désormais des appels téléphoniques VoIP entrants via Session Initiation Protocol — un numéro de téléphone peut donc être directement routé vers un agent gpt-realtime.

Côté développeur : les reusable prompts permettent de sauvegarder une configuration agent complète (prompt système, tools, variables, exemples de turns user/assistant) et de la réutiliser across sessions via un simple identifiant. Ça réduit le setup time et la taille des payloads pour les agents en production. La Realtime API était en beta depuis l'automne 2025 ; la GA inclut des SLAs et un support enterprise.

🛠 Pourquoi c'est à connaître pour Lynxter

La combinaison SIP + MCP est la nouveauté la plus concrète pour le support Lynxter. Imagine un numéro de téléphone dédié "support S300X/S600D" : quand un client appelle, l'agent gpt-realtime répond, accède à l'historique des tickets via un connecteur MCP, triage le problème (erreur de calibration ? défaut matériel ? formation utilisateur ?), et transfère à un humain avec un résumé préchargé si nécessaire. Ce n'est pas un chantier de cette semaine, mais c'est maintenant une plomberie API standard — pas de la R&D. Le budget de benchmarking est justifié.

Le point de comparaison pour Lynxter reste Voxtral de Mistral (CC BY-NC 4.0 sur les open weights, $0.016/1k chars en API) pour la voix française, et l'API Claude pour la logique métier. Une architecture hybride — Voxtral pour la voix, Claude via MCP pour le raisonnement — pourrait être plus économique et potentiellement on-prem si un client industriel demande de la souveraineté. La GA OpenAI fixe le pricing de référence : à comparer à Mistral Studio pour un équivalent français.
