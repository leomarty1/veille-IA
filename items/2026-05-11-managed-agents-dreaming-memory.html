Managed Agents — Dreaming, multiagent orchestration, memory

Extrait du brief Anthropic consolide, Mistral arme — 2026-05-11

Contexte étendu

Anthropic publie une suite de fonctionnalités regroupées sous le terme Managed Agents, qui adressent toutes le même problème : les agents long-running. Trois pièces :

Dreaming (research preview) — les agents continuent de bosser entre les sessions actives. Concrètement, un agent peut être laissé sur un problème complexe et continuera d'explorer en background pendant que tu fais autre chose, puis te présentera ses conclusions à ta prochaine session. C'est l'évolution naturelle de l'agentic loop classique vers un mode "background worker".

Multiagent orchestration — framework natif pour faire collaborer plusieurs agents Claude avec des rôles différents (researcher / coder / reviewer par exemple). Concurrent direct du Claude Agent SDK custom et des frameworks tiers (LangGraph, AutoGen).

Memory en public beta — état persistant entre sessions, indexé et requêtable. Évolution majeure par rapport à la mémoire ChatGPT car structurée et auditable (cf. transparence "Memory sources" côté OpenAI). Activable / désactivable / éditable par l'utilisateur.

🎯 Pourquoi c'est important pour Lynxter

Pas immédiatement à déployer mais l'infrastructure de référence pour la prochaine étape de l'automation Lynxter. Le scénario qui se dessine à 6-12 mois (cf. prédiction n°4) :

Un agent "Lynxter Support Background" qui :
1. Memory — connaît l'historique de chaque client (machine, mat'eriaux, problèmes passés)
2. Dreaming — surveille en continu les tickets, fait du pré-diagnostic en background
3. Multiagent — un agent diagnostic + un agent rédaction réponse + un agent escalade, qui collaborent

À ce stade c'est encore expérimental, mais les briques sont là. Action concrète : noter mentalement que memory est en public beta, jouer avec en sandbox pour comprendre le modèle mental avant d'en faire dépendre un workflow critique.
