Contexte étendu

Anthropic livre Claude Opus 4.7 en disponibilité générale avec un saut net sur le coding agent — 87.6 % sur SWE-bench Verified contre 80.8 % pour Opus 4.6, et 64.3 % sur SWE-bench Pro contre 53.4 %. C'est la première fois que le modèle franchit la barre symbolique des 85 % sur Verified, et il prend la tête devant tous les concurrents publics (GPT-5.5 Instant n'est pas évalué sur ce benchmark, Gemini 3 Pro est estimé à ~82 %, Mistral Medium 3.5 à 77.6 %).

Sur les nouveautés produit : Anthropic introduit un niveau d'effort "xhigh", intermédiaire entre high et max, pour offrir un cran de plus aux tâches qui méritent un budget de raisonnement étendu sans aller jusqu'au max. Le support des images haute-résolution passe à 2576px (3.75 MP), suffisant pour analyser des screenshots de documents techniques en pleine définition. Et surtout — fenêtre 1M tokens accessible au tarif standard, sans le premium long-context jusqu'ici facturé séparément.

Le pricing reste à $5 par million de tokens en input et $25 en output — inchangé par rapport à Opus 4.6 malgré le saut de capacités. Distribué sur l'API Claude, AWS Bedrock, Google Vertex et Microsoft Azure Foundry simultanément.

🎯 Pourquoi c'est important pour Lynxter

Pour le workflow Lynxter, ce saut SWE-bench se traduit concrètement par moins d'allers-retours sur les tickets support qui impliquent de comprendre ou modifier du code — typiquement les intégrations Cowork, les skills custom, ou les scripts d'analyse logs. L'écart de 7 points sur Verified n'est pas cosmétique : il correspond à la zone où le modèle commence à résoudre en autonomie des bugs que Opus 4.6 abandonnait à mi-chemin.

Le niveau d'effort xhigh est probablement le plus utile au quotidien : entre les tâches rapides (high) et les analyses critiques (max), Léo pourra cibler le bon trade-off pour la majorité des sessions support. Et le 1M tokens sans premium supprime le besoin de stratégies de chunking dans les skills custom — un brief client de 200 pages tient désormais dans une session sans découpage manuel.

Action concrète : audit des prompts existants du plugin lynxter-support pour identifier ceux qui bénéficieraient du xhigh effort. Probablement les /diagnostic et /cr-intervention qui demandent du raisonnement multi-source.
