Fable 5 : record à 95 %,
coupé en trois jours

Semaine charnière. Le 9 juin, Anthropic rend public Claude Fable 5 — les mêmes poids que le Mythos 5 jusque-là réservé à Project Glasswing — qui établit un record pour un modèle grand public : 95,0 % sur SWE-bench Verified (vs 88,7 % pour GPT-5.5) et 80,3 % sur SWE-bench Pro (vs 69,2 % pour Opus 4.8). Trois jours plus tard, le 12 juin, une directive de contrôle à l'export du gouvernement américain force Anthropic à couper Fable 5 et Mythos 5 pour tout le monde — premier cas connu d'un modèle frontière commercial retiré du marché par l'État. En parallèle, Claude Code enchaîne huit versions (sous-agents imbriqués sur 5 niveaux, --safe-mode, enforceAvailableModels), OpenAI rachète Ona (ex-Gitpod) pour des agents Codex longue durée, et l'open-weights de code accélère avec DiffusionGemma (Google) et North Mini Code (Cohere). Meta et Mistral : rien de produit dans la fenêtre.

TL;DR

  • Claude Fable 5 lancé puis coupé en 3 jours. Le 9 juin, Anthropic ouvre au public Fable 5 (mêmes poids que Mythos 5) : 95,0 % SWE-bench Verified — record pour un modèle grand public, +6,3 pts sur GPT-5.5 — et 80,3 % SWE-bench Pro (+11,1 pts sur Opus 4.8), 1M tokens en entrée / 128K en sortie, adaptive thinking always-on, à $10/$50 par M tokens. Le 12 juin, une directive d'export-control US suspend l'accès pour tout ressortissant étranger ; ne pouvant filtrer en temps réel, Anthropic coupe les deux modèles pour tous. Les autres modèles (Opus 4.8…) restent en ligne.
  • Claude Code passe à l'orchestration imbriquée. Huit versions en cinq jours (2.1.169 → 2.1.176) : les sous-agents peuvent spawner leurs propres sous-agents jusqu'à 5 niveaux de profondeur (2.1.172), nouveau flag --safe-mode qui désactive toutes les customisations, et setting enforceAvailableModels qui verrouille le modèle Default dans une allowlist managée — gouvernance de flotte et de coûts. Intégration de Fable 5 livrée en 2.1.170, avec auto-fallback vers Opus 4.8.
  • OpenAI rachète Ona, l'open-weights de code accélère. OpenAI acquiert Ona (ex-Gitpod) pour faire tourner Codex en sandbox cloud des heures voire des jours sans surveillance — 5 M d'utilisateurs Codex/semaine, +400 % depuis janvier. Côté open weights : DiffusionGemma (Google, 26B MoE texte par diffusion, 1 000+ tok/s sur 1 H100, Apache 2.0) et North Mini Code (Cohere, 30B/3B, 80,2 % pass@10 SWE-bench, tourne sur 1 H100). OpenAI retire aussi GPT-5.2 de ChatGPT (bascule auto GPT-5.5).

🎯 Implications Lynxter

La leçon Fable 5, ce n'est pas le 95 % — c'est la coupure. Sur le papier, Fable 5 est le meilleur outil possible pour ton terrain le plus dur : 80,3 % SWE-bench Pro (+11,1 pts vs Opus 4.8, +21,7 pts vs GPT-5.5), le benchmark des tâches longues, multi-fichiers, mal spécifiées — exactement un audit firmware S300X/S600D ou une migration de scripts Cowork. Mais deux faits commandent la prudence : son pricing à $10/$50 par M tokens (2× le standard Opus 4.8 à $5/$25), et surtout sa suspension le 12 juin, trois jours après le lancement, par directive d'export-control. Conclusion opérationnelle : ne fais dépendre aucun workflow critique ou client de Fable 5. Garde tes Routines Cowork et Claude Code épinglées sur Opus 4.8 (à $5/$25, non affecté par la directive). Si Fable 5 revient, traite-le comme un upgrade expérimental à A/B-tester sur des tâches à forte valeur (audit, migration), pas comme une brique de prod.

Claude Code te donne enfin la gouvernance de flotte qui manquait. Deux settings de la semaine comptent pour Lynxter. enforceAvailableModels verrouille le modèle Default dans une allowlist managée : les settings utilisateur/projet ne peuvent plus élargir la liste — tu peux donc imposer Opus 4.8 (et interdire le premium hors budget) à toute l'équipe depuis une config centrale, et le fallback automatique vers Opus 4.8 couvre les orgs sans accès au modèle haut de gamme. --safe-mode (et l'env CLAUDE_CODE_SAFE_MODE) désactive toutes les customisations pour des runs reproductibles — utile pour débugger une routine qui dérape. Et les sous-agents imbriqués sur 5 niveaux (2.1.172) débloquent une orchestration plus profonde : la routine veille elle-même fait déjà du fan-out par acteur, on peut maintenant imbriquer recherche → croisement → rédaction sans saturer le contexte de l'orchestrateur. Action cette semaine : pose enforceAvailableModels avec Opus 4.8 en Default, et audite quelles routines spawnent des sous-agents pour mesurer le coût réel via la nouvelle attribution d'usage VSCode.

L'open-weights de code devient une vraie option de souveraineté — et l'épisode Fable 5 le justifie. Deux sorties cette semaine changent la donne pour tes clients industriels qui ne peuvent pas envoyer leur code/données vers un cloud US (défense, aéro, médical réglementé) : North Mini Code de Cohere (MoE 30B/3B actifs, Apache 2.0, 256K de contexte, 80,2 % pass@10 / 61,0 % pass@1 SWE-bench Verified, tourne sur un seul H100) et DiffusionGemma de Google (26B MoE par diffusion, 1 000+ tok/s sur 1 H100, Apache 2.0, mais explicitement expérimental). Quand un modèle frontière peut être coupé par décret en trois jours, un modèle open-weights déployé on-prem n'a, lui, aucun fournisseur capable de te débrancher. Action : benchmarke North Mini Code contre Devstral sur une tâche Lynxter réelle (migration de scripts de calibration, génération de gcode-helpers) ; s'il tient à 60-80 % de réussite sur un H100 que tu contrôles, tu as un argumentaire RFP « IA souveraine sur site » à présenter aux clients sensibles.

Anthropic rend public le 9 juin Claude Fable 5, qui déploie les mêmes poids que Mythos 5 (jusque-là réservé à Project Glasswing). Washington débranche Fable 5 et Mythos 5 — première suspension d'un modèle frontière par l'État

🎯 Lynxter 2026-06-12

Moins de trois jours après le lancement, le 12 juin, le gouvernement américain émet une directive de contrôle à l'export invoquant la sécurité nationale, interdisant l'accès à Fable 5 et Mythos 5 à tout ressortissant étranger — y compris hors US et y compris les employés étrangers d'Anthropic. Ne pouvant filtrer les ressortissants étrangers en temps réel, Anthropic coupe les deux modèles pour tout le monde ; les autres modèles (Opus 4.8, Sonnet, Haiku) restent en ligne. Motif invoqué : la découverte supposée d'une méthode de jailbreak de Fable 5. Anthropic conteste publiquement qu'un jailbreak étroit justifie le retrait d'un modèle commercial, avertissant qu'un tel standard « halterait tout déploiement de modèle frontière ». C'est le premier cas connu d'un modèle frontière commercial retiré du marché par directive d'export US.

Claude Code 2.1.169–176 — sous-agents imbriqués sur 5 niveaux, --safe-mode, enforceAvailableModels

Depuis le 12 juin, GPT-5.2 Instant, Thinking et Pro ne sont plus disponibles dans ChatGPT ; les conversations existantes basculent automatiquement sur le modèle GPT-5.5 correspondant. DiffusionGemma — modèle texte par diffusion 26B open weights, jusqu'à 4× plus rapide

🛠 Utile 2026-06-10

Google sort le 10 juin DiffusionGemma (diffusiongemma-26B-A4B-it), un MoE 26B total / 3,8B actifs bâti sur le backbone Gemma 4 mais qui génère le texte par diffusion discrète : il denoise des blocs de 256 tokens en parallèle au lieu d'un token autorégressif à la fois. Débit : 1 000–1 100+ tokens/s sur un seul H100 (FP8), 700+ sur RTX 5090, annoncé jusqu'à 4× plus rapide que l'équivalent autorégressif. Contexte 256K, ~18 Go de VRAM quantifié, licence Apache 2.0, dispo day-zero sur Hugging Face, Kaggle et Vertex AI. Caveat majeur : Google le qualifie d'expérimental, avec des scores nettement inférieurs à Gemma 4 en raisonnement, maths et code.

developers.googleblog.com — DiffusionGemma guide · marktechpost.com
