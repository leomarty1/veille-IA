TL;DR

  • Quatorze releases Claude Code (2.1.202→215) en deux semaines. 2.1.207 bascule le défaut sur Opus 4.8 pour Bedrock/Vertex/AWS et active l'auto mode sans opt-in ; 2.1.212 refond l'orchestration (/fork en session de fond, /subtask pour l'ancien comportement, cap 200 subagents/session, MCP >2 min en arrière-plan, 200 WebSearch/session) ; 2.1.214 ajoute EndConversation et colmate des trous de permissions (plan mode lançant des Bash mutants, règles dir/** trop larges) ; 2.1.215 impose l'invocation explicite de /verify et /code-review.
  • Le paysage benchmarks se recompose. OpenAI passe GPT-5.6 en GA (Sol $5/$30, Terra $2,50/$15, Luna $1/$6 ; Sol Ultra 91,9 % Terminal-Bench vs Opus 4.8 78,9 %) ET audite SWE-Bench Pro — ~30 % des 731 tâches jugées cassées, recommandation d'usage retirée : les scores SWE-bench des slides/RFP sont à relativiser. Meta rompt avec l'open weights via Muse Spark 1.1 (closed API, 1,25 $/4,25 $ par M tokens, MCP Atlas 88,1 devant Opus 4.8 82,2), xAI sort Grok 4.5 (Intelligence Index 54, 4e ; 2 $/6 $).
  • Sécurité des agents au premier plan. xAI open-source Grok Build sous Apache 2.0 (~72 h après la preuve qu'une version du CLI exfiltrait des dépôts Git complets et des clés SSH vers un bucket xAI) — à comparer à Claude Code et Codex qui n'exfiltrent pas par défaut. Perplexity dévoile SPACE (microVM Firecracker, injection de credentials hors sandbox), Cursor 3.11 distribue des Team MCP servers vers IDE/CLI/cloud, et OpenAI durcit GPT-5.6 avec GPT-Red (6× moins d'injections de prompt réussies).

🎯 Implications Lynxter

La séquence Claude Code 2.1.202→215 impose deux actions cette semaine. Sur les déploiements cloud d'abord : la 2.1.207 bascule le défaut sur Opus 4.8 et active l'auto mode sans opt-in sur Bedrock, Vertex et Claude Platform on AWS — audite tes variables d'environnement avant toute montée de version sur ces clouds, car un pipeline qui écrit dans un repo firmware S300X/S600D peut passer en auto mode par simple upgrade. Sur les routines Cowork ensuite : la 2.1.215 met fin à l'exécution implicite de /verify et /code-review. Tout workflow qui comptait dessus ne les lancera plus — ajoute les appels explicites sans attendre, sous peine de merges non vérifiés qui passent en silence. Et dimensionne tes workflows fan-out sous le nouveau cap de 200 subagents/session, en migrant les usages in-session de /fork vers /subtask.

La sécurité des agents devient le sujet dominant, et il tombe au bon moment pour trancher la gouvernance interne. L'incident Grok Build — une version du CLI xAI qui exfiltrait des dépôts Git complets et jusqu'aux clés SSH du poste vers un bucket xAI — prouve que le périmètre à risque n'est pas le repo mais tout le home de la machine. Établis une allowlist des CLI d'agents autorisés sur les postes dev/support (Claude Code, Codex, Grok Build, Cursor CLI…), documente leur politique de rétention par défaut, et isole toute exécution non vérifiée hors des clés de production. Le modèle d'isolation de Perplexity SPACE (microVM Firecracker éphémère, credentials injectés depuis l'extérieur du sandbox au dernier moment) est la référence architecturale à répliquer dès que des agents long-horizon touchent des données machine sensibles. À l'inverse, le durcissement de permissions de Claude Code 2.1.214 (fin des Bash mutants en plan mode, resserrement des règles dir/**) renforce l'argument « Claude Code est le CLI d'agent le plus verrouillé par défaut ».

Côté modèles, trois sorties concurrentes sont à benchmarker, pas à adopter les yeux fermés — et l'audit SWE-Bench Pro d'OpenAI oblige à en relativiser les chiffres. GPT-5.6 GA (Terra à $2,50/$15, Sol Ultra 91,9 % Terminal-Bench), Muse Spark 1.1 (closed API à 1,25 $/4,25 $, MCP Atlas 88,1 devant Opus 4.8 à 82,2) et Grok 4.5 (Coding Agent Index 76, ~4,2× moins de tokens qu'Opus 4.8) cassent tous la grille de prix, mais chacun porte un défaut opposable : Muse Spark est une closed API sans self-host (dépendance fournisseur totale), Grok 4.5 hallucine 54 % du temps et n'est pas dispo dans l'UE. Puisqu'OpenAI reconnaît lui-même que ~30 % des tâches SWE-Bench Pro sont cassées, ajoute sur la page Modèles et dans tes RFP une note « benchmark contesté (juillet 2026) » et déplace l'argumentaire vers le vrai discriminant : un test coût/fiabilité sur un corpus S300X/S600D représentatif, pas un score public.

OpenAI audite SWE-Bench Pro et retire sa recommandation — ~30 % des tâches cassées

🛠 Utile 2026-07-08

Dans Separating signal from noise in coding evaluations (8 juil.), OpenAI juge ~30 % des 731 tâches publiques de SWE-Bench Pro cassées, en quatre catégories : tests trop stricts rejetant des solutions fonctionnellement correctes, specs vagues masquant les attentes des tests, tâches superficielles laissant passer des solutions incomplètes, descriptions trompeuses orientant l'agent vers le mauvais code. Conclusion : le benchmark ne mesure plus fiablement la capacité de coding frontière, recommandation d'usage retirée, accompagnée d'un post séparé sur les limites de SWE-bench Verified. Implication directe : les scores SWE-bench cités dans slides/RFP (dont ce brief et la page Modèles du site) sont à relativiser.

· Info 2026-07-15

Grok Build passe en open source (Apache 2.0) 72 h après une exfiltration silencieuse de dépôts Git et clés SSH

🎯 Lynxter 2026-07-15

xAI open-source Grok Build — son agent de code CLI/TUI en Rust (99,6 %), harness d'~844 530 lignes selon Simon Willison, avec skills/plugins/hooks/serveurs MCP/subagents et Agent Client Protocol — sous Apache 2.0 le 15 juil. (github.com/xai-org/grok-build, ~20,3k stars, contributions externes refusées). Le geste survient ~72 h après des preuves wire-level qu'une version du CLI uploadait silencieusement des dépôts Git complets vers un bucket cloud contrôlé par xAI — un utilisateur l'ayant lancé dans son home rapportant l'envoi de ses clés SSH, base de mots de passe, documents, photos et vidéos. xAI a désactivé la rétention par défaut le 12 juil. et supprimé les données retenues ; le binaire peut désormais tourner local-first. À comparer à Claude Code et OpenAI Codex, qui n'exfiltrent pas les repos par défaut.

Grok 4.5 — 4e de l'Intelligence Index à 54, 76 au Coding Agent Index, 2 $/6 $ par M tokens, co-entraîné avec Cursor

🛠 Utile 2026-07-08

xAI sort Grok 4.5 le 8 juil., premier modèle maison pensé code/agentique, co-entraîné avec Cursor sur des trillions de tokens d'interactions dev/agent. Artificial Analysis le classe 4e de l'Intelligence Index à 54, derrière Fable 5 (60), Opus 4.8 (56) et GPT-5.5 (55), soit +16 pts vs Grok 4.3 ; il marque 76 au Coding Agent Index (parité GPT-5.5 xhigh sous Codex, juste sous Fable 5 max sous Claude Code) et 64,7 % sur SWE-bench Pro. Perplexity SPACE — runtime sandbox sécurisé pour agents long-horizon (microVM Firecracker, spin-up 3-5×, isolation credentials)

🛠 Utile 2026-07-15

Perplexity dévoile SPACE (Secure and Efficient Runtimes for Long-Running Agents, 15 juil.), la couche runtime de son agent Perplexity Computer. Chaque tâche tourne dans une microVM Firecracker éphémère, détruite en fin de tâche mais encapsulée dans une session pausable/resumable/forkable. La sécurité repose sur trois services séparés — Network Gateway, Space Daemon, File Service — qui injectent les credentials depuis l'extérieur du sandbox uniquement au moment du besoin : un agent compromis n'a par défaut aucun chemin vers les credentials plateforme. Perplexity revendique un spin-up 3-5× plus rapide et, sur la 1re semaine, des millions de créations de sandbox et des dizaines de millions de reconnexions. Même problème d'isolation que les code-execution sandboxes de Claude Code (bash) et OpenAI, packagé en couche runtime dédiée.

siliconangle.com · aibusiness.com
