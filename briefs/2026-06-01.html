🎯 Implications Lynxter

Opus 4.8 est un upgrade silencieux et immédiat de ton stack. Anthropic livre 4.8 à même pricing que 4.7 ($5/$25 par M tokens), donc l'arbitrage est trivial : tu actualises tes Routines Cowork et Claude Code, point. Le saut de +1 pt sur SWE-bench Verified (87.6 → 88.6 %) reste discret face à GPT-5.5 (88.7 %), mais le vrai signal est ailleurs : +4.9 pts sur SWE-bench Pro (64.3 → 69.2 %), soit 10.6 pts d'avance sur GPT-5.5 sur le benchmark le plus représentatif des tâches longues, multi-fichiers, mal spécifiées — exactement le profil d'un audit firmware S300X ou d'une migration de scripts Cowork. Le fast mode divise par 3 son surcoût vs la 4.7 fast mode : il devient économiquement viable pour les batch jobs où la latence prime sur la qualité absolue (scan tickets entrants, déduplication base de connaissance, classification logs). Test concret cette semaine : lance une routine Cowork existante en fast mode, mesure la dérive qualité — si elle reste sous 10 %, bascule les workloads non-critiques pour réduire le wall-clock d'un facteur 2.5.

Dynamic Workflows change le plafond de ce que tu peux orchestrer depuis Claude Code. Jusqu'ici, une routine multi-agent saturait vite : la fenêtre de contexte du Claude orchestrateur se remplit des résultats intermédiaires, et passé 5-6 sous-agents la qualité décroche. Les Dynamic Workflows déplacent le plan dans un script JS exécuté par un runtime séparé — Claude écrit le script (ou tu réutilises le bundled /deep-research), le runtime fan-out jusqu'à 1 000 agents au total dont 16 concurrents, et seul le rapport final atterrit dans ton contexte. Pour Lynxter, c'est le déblocage du brief veille-IA lui-même : la routine actuelle se contente d'un orchestrateur séquentiel par acteur. En Dynamic Workflow, on parallélise : un sous-agent par acteur (Anthropic, OpenAI, Google, Meta, Mistral, Perplexity, xAI, Cursor) qui fetch + résume en parallèle, un agent de cross-check qui élimine les doublons, un agent rédacteur final. Estimation wall-clock : ~3 min vs 12-15 min actuellement. Active /effort ultracode dans une session test la semaine prochaine pour voir Claude planifier automatiquement les workflows. Requis : Claude Code v2.1.154+. Le mot-clé workflow dans un prompt déclenche aussi un run ad-hoc — utile pour les audits ponctuels firmware S300X/S600D ("workflow audit every parameter set in /firmware/profiles for inconsistencies").

Le passage Anthropic devant OpenAI sur la valo recadre tes paris fournisseur à 3 ans. $965 Md post-money, $47 Md de run-rate, ARR triplé depuis février ($380 Md → $965 Md). La signature : un IPO probable dans les 12-18 mois et un appétit confirmé sur l'enterprise (run-rate cross $47 Md). Pour Lynxter, cela conforte le pari Claude comme stack principal : pas de risque crédible de discontinuité fournisseur, capacité d'investissement R&D ≥ OpenAI sur les 24 mois à venir, et roadmap Mythos (preview 93.9 % SWE-bench, dropée incidemment dans l'écosystème) qui suggère Claude 5 avant fin 2026. À surveiller : la pression IPO pourrait pousser une monétisation plus agressive (paliers Cowork enterprise, plus de feature-gating sur les Routines). Si tu prévois une montée en charge support client via Claude Cowork au S2 2026, négocie un commit annuel maintenant pour figer le pricing.

Dynamic Workflows en research preview — Claude Code orchestre jusqu'à 1 000 sous-agents

🎯 Lynxter 2026-05-28

Anthropic introduit les Dynamic Workflows dans Claude Code (research preview, v2.1.154+, tous plans payants y compris Pro). Un workflow est un script JavaScript que Claude écrit pour ta tâche et qu'un runtime exécute en arrière-plan — la session reste réactive pendant que les agents bossent. Le runtime supporte jusqu'à 16 sous-agents en parallèle, 1 000 au total par run. Le mot-clé workflow dans un prompt déclenche un run ad-hoc ; le mode /effort ultracode planifie automatiquement des workflows pour chaque tâche substantielle de la session. Workflow bundled fourni : /deep-research (fan-out web search, fetch et cross-check de sources, rapport cité avec claims non vérifiés filtrés). Les runs sont sauvegardables comme commande personnalisée (.claude/workflows/ projet ou ~/.claude/workflows/ global), pausables, resumables dans la même session. Anthropic vante Jarred Sumner ayant porté Bun de Zig à Rust (750 k lignes de code) via cette primitive.

code.claude.com/docs/en/workflows · anthropic.com/news

Cursor 3.6 — Auto-review (classifier subagent) + Build in Parallel (graphe de dépendances)

🛠 Utile 2026-05-29

Release Cursor 3.6 le 29 mai. Auto-review : nouveau run mode pour les tâches longues avec moins de prompts d'approbation. S'applique à Shell, MCP, et Fetch tool calls. Les appels allowlistés tournent immédiatement, ceux qui peuvent être sandboxés tournent en sandbox, les autres passent par un classifier subagent qui décide d'autoriser, retenter autrement, ou demander approbation. Build in Parallel : Cursor analyse les dépendances entre les étapes d'un plan multi-step, construit un graphe d'exécution et dispatch les étapes indépendantes simultanément à des sous-agents async. Split PRs : action rapide pour découper les changements en PRs logiques. Composer 2.5 élargi avec meilleure stabilité long-tasks. Modèle de billing Bugbot bascule en usage-only (suppression seat fees). Cursor reste le concurrent direct de Claude Code + Composer côté IDE multi-agent — la convergence agentic IDE est désormais établie (Claude Code, Cursor, Antigravity, Codex).

xAI lance Custom Skills sur Grok — workflows persistants, à la Anthropic Skills

CNN poursuit Perplexity — neuvième litige copyright actif

· Info 2026-05-28

CNN dépose plainte contre Perplexity pour copie et redistribution illégale de son contenu, après échec des négos de licensing l'an dernier. Avec ce dépôt, Perplexity est désormais visé par 9 actions actives : CNN, New York Times, News Corp + Dow Jones, New York Post, Chicago Tribune, Encyclopedia Britannica, Merriam-Webster, Reddit, Yomiuri Shimbun. À l'opposé : Time, Gannett, Le Monde, Der Spiegel ont signé des accords de licensing plutôt que litiger. Croissance Perplexity solide en parallèle (50 % MoM en mars, 100 M MAU avec produits agents inclus). Signal de fond : le modèle "answer engine" sans licensing arrive à son plafond légal — soit accord soit procès.
