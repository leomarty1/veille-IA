TL;DR

  • Google I/O 2026 (19 mai) sort Antigravity 2.0 — desktop, CLI Go, SDK et Managed Agents dans l'API Gemini — premier rival sérieux à Claude Code sur le terrain agent CLI, alimenté par Gemini 3.5 Flash ($1.50/M in, $9/M out, bat Gemini 3.1 Pro sur Terminal-Bench 2.1 à 76.2 %).
  • Anthropic ouvre la Compliance API (21 mai) avec 28 partenaires DLP/SIEM (Palo Alto, Proofpoint, SailPoint, Tenable) — gouverner Claude Enterprise comme on gouverne les autres SaaS, signal d'agressivité commerciale vers les boîtes régulées.
  • Meta licencie 8 000 personnes (≈10 % de l'effectif) le 20 mai et réoriente 7 000 vers Applied AI / Agent Transformation Accelerator — CapEx 2026 confirmé à 115-135 milliards $, presque le double de 2025.

🎯 Implications Lynxter

Antigravity 2.0 est le premier challenger sérieux à Claude Code sur ton terrain. Google livre en une seule annonce ce que Anthropic a mis 18 mois à construire : un desktop app orienté orchestration multi-agents (subagents qui se spawn en parallèle, jusqu'à 93 sous-agents dans une démo OS-in-12h), une CLI Go pour les workflows headless, un SDK pour custom agents, et une Managed Agents API qui spin-up un agent en isolation Linux en un appel. Le tout alimenté par Gemini 3.5 Flash à $1.50/M input — soit 5-6× moins cher que Claude Opus 4.7 ($15/M input). Pour Lynxter, c'est un benchmark concret à faire dans les semaines qui viennent : prendre une tâche Cowork qui tourne actuellement sur Opus 4.7 (génération brief veille-IA, scan tickets support S300X), la rejouer en parallèle sur Antigravity + Gemini 3.5 Flash, mesurer qualité de sortie et coût total. Si l'écart qualité < 15 % et coût ÷5, le hybrid stack (Claude pour l'orchestration critique, Antigravity pour les batch low-stakes) devient une option économique majeure.

La Compliance API change la donne pour le passage Lynxter → Claude Enterprise. Tant que le stack reste sur Claude Pro/Max, la gouvernance se limite aux paramètres workspace. Avec la Compliance API (Claude Enterprise uniquement), un SailPoint ou Proofpoint peut maintenant logger toutes les conversations, appliquer DLP sur les uploads fichiers, et tracer les actions admin — ce qui débloque la conformité requise pour exposer Claude au support client (réponses tickets S300X/S600D vues par des opérateurs externes). Si Lynxter envisage à terme un agent vocal support entrant ou un workflow CRM partagé avec un prestataire, le bascule vers Enterprise + un partenaire DLP devient le chemin standard. À mettre dans la roadmap T4 2026 si la croissance support le justifie.

Claude Code 2.1.147-150 apporte des QoL utiles tout de suite. Le /code-review remplace /simplify (annonce-le aux opérateurs avant qu'ils ne galèrent), /usage liste maintenant la conso par catégorie (skills, subagents, plugins, MCP servers) — précieux pour identifier quel plugin Cowork consomme tes crédits Max. La rewind menu gagne "Summarize up to here" pour compresser le contexte ancien en gardant les derniers tours intacts : utile sur les longues sessions support qui accumulent du contexte ticket. Pas de blocker, mais 30 minutes d'audit des Routines Lynxter pour profiter des nouveautés.

Un modèle OpenAI réfute la conjecture d'Erdős sur les distances unité (≈80 ans ouverte)

· Info 2026-05-20

Un modèle interne OpenAI fournit une famille infinie de constructions qui battent polynomialement l'estimation grille-carrée pour le problème des distances unité (Erdős 1946 : "n points dans un plan, combien de paires à distance 1 max ?"). Vérifié indépendamment par 9 mathématiciens (Timothy Gowers — médaille Fields, Noga Alon, Thomas Bloom, Will Sawin, Melanie Wood, Daniel Litt, Arul Shankar, Jacob Tsimerman, Victor Wang) qui co-signent un companion paper. Construction utilise la théorie algébrique des nombres. Signal sur la maturité des reasoning models en mathématiques pures — au-delà des Olympiades, sur des problèmes ouverts depuis des décennies.
